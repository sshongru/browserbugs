BUG: autocapitalize doesn't work on content editable regions

The documentation seems to imply that this is only available on form elements like input and textarea. But this proposal from Apple in 2011 makes it look like it should be added to contenteditable regions too... maybe there's a chance this is coming eventually?

Environment

iPad Air with iOS 8.1.2

Steps to Reproduce:

  1. Using an iPad with iOS 8...
  2. Tap into a red region
  3. Type in a sentence
  4. Notice the contenteditable region doesn't respect the "words" autocapitalize property
  5. Also notice that in the form fields the second character of each word is being capitalized
