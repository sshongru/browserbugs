Bold text cannot be unbolded when it uses a font weight under 600

It seems like the browser is deciding that since the b/strong tags have a weight closer to normal than bold that the text is already "unbolded" even though visually and semantically that isn't the case. The browser should instead decide whether to unbold something based on the semantic tags involved in the selection.

Steps to Reproduce:

  1. Using Chrome/Safari/Firefox...
  2. Select some bold text in the following red contenteditable region
  3. Click the "Toggle Bold" button (or the CTRL+B shortcut in some browsers)
  4. Notice that the text doesn't unbold

Select this bold text then click "Toggle Bold" to unbold it. Notice nothing happens in Chrome/Safari/Firefox. IE11/Edge work correctly.
