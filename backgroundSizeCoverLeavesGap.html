Chrome: Images of certain dimensions dont fully cover the background of elements that are 60x60

Steps to reproduce:

  1. Launch this page in the Chrome browser on a retina screen Macbook
  2. Notice the yellow line on the top two images

Expected Results:
No yellow line

Actual Results:
Yellow line

It seems the background-size: cover CSS property does not fully cover the background of an element when the image used is of certain dimensions and the element is 60x60pixels. This page demonstrates it happening with 201x201 and 276x276 PNG images. Interestingly enough if you change the size of the element to 59x59 then the two failing cases start passing and the passing case starts failing.

Reproduced in Chrome 57.0.2987.133 (64-bit) on retina Macbook Pro. Works fine in Firefox and Safari.
