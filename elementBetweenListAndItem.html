IE: Ordered list items are all 1s when element between li and ol exists

Steps to reproduce

  1. Place a div tag between an ol and li element
  2. Notice in IE and Edge that the list numbers always start with 1

Notes

According to MDN the only tags permitted inside a ol are "Zero or more <li> elements" so it is a little odd to be doing this. Every other browser seems to handle it fine, it's just IE/Edge that trip up.

I believe Microsoft has said that any departure from webkit rendering in Edge is considered a bug so I would definitely consider this a bug in Edge.

Example

  1. One
  2. Two
  3. Three
  4. Four
  5. Five
  6. Six

Workaround

Manually set each li tag's value property.
